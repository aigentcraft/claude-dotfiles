# UC: パターンの局所実装 — 同種構造への横展開が発生しなかった

## Type
User Correction — `uc-local-pattern-no-generalization`
**Date**: 2026-02-28
**Cluster**: `ai-behavior`

---

## ユーザーの指摘

> なぜ、Graphragを作ることがわかっていながら、今回のようなことが起こったのか。
> また、それについては思考し、ナレッジとして蓄積できているのか。

---

## 何が起きたか

`skills-graph/relationships.md`（スキル間エッジ定義）を作成した際、
`error-graph/relationships.md` は作られなかった。

- 「グラフ構造にはエッジ定義が必要」というパターンを知っていた
- `skills-graph` ではそれを実装した
- しかし `error-graph` という同種の構造に横展開しなかった

結果: `error-graph` はノードとクラスターだけの「文書集合」になり、グラフとして機能しなかった。

---

## 根本原因

**パターンを「そのユースケース専用の実装」として扱い、「構造の抽象的要件」として認識しなかった。**

```
正しい認識: 「グラフ構造 → 必ずエッジ定義が必要」（汎用要件）
実際の認識: 「skills-graph → スキル間補完関係の定義が必要」（局所要件）
```

この差が横展開の自問を消した。

## 正しい考え方

パターンをある場所に実装したとき、必ず問う:

> **「このプロジェクト内で、同種の構造が他にないか？」**

```
実装した構造       → 同種の構造を探す
relationships.md  → error-graph, skills-graph, ... すべてのグラフ構造
PROJECT_MAP.md    → すべてのプロジェクト
エッジタイプ定義   → すべてのグラフノード追加時
```

---

## 予防ルール（横展開チェック）

**新しいパターンを実装したら即座に自問する:**

1. この実装は「このファイル専用のルール」か、「この種の構造すべてに必要な要件」か？
2. 同種の構造が他にいくつある？（Glob で確認）
3. それぞれに同じパターンが適用されているか？
